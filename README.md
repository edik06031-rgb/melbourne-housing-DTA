# melbourne-housing-DTA

**Тип задачі:** Регресія (прогноз числової змінної `Price`)  
**Джерело даних:** [Melbourne Housing Snapshot, Kaggle](https://www.kaggle.com/datasets/dansbecker/melbourne-housing-snapshot)  
**Датасет:** 13 580 рядків, 21 ознака


### 1. Очищення даних
- Поле `Date` перетворено на тип datetime (`dayfirst=True`).
- Нулі у `Landsize` та `BuildingArea` замінено на `NaN` (нуль фізично неможливий).
- `YearBuilt` < 1800 → `NaN` (явна помилка вводу; виявлено значення 1196).
- Дублікатів рядків не знайдено.
- Стовпці `Address`, `SellerG`, `Postcode` видалено як нерелевантні.

### 2. Аналіз (EDA)
- Розподіл ціни **правоскошений**: медіана ≈ 903 000, середнє ≈ 1 076 000 AUD.
  Логарифм ціни ближчий до нормального.
- Будинки (`h`) суттєво дорожчі за юніти (`u`) і таунхауси (`t`).
- Найбільша кореляція з ціною: `Rooms` (+0.50), `Bathroom` (+0.47), `YearBuilt` (-0.33).
- `Rooms` і `Bedroom2` корелюють між собою (r = 0.94) → залишили лише `Rooms`.

### 3. Нові ознаки (без витоку)
| Ознака | Опис |
|---|---|
| `sale_year`, `sale_month` | Рік і місяць продажу |
| `age` | Вік будинку на момент продажу |
| `has_buildingarea` | Прапорець: чи відома площа будівлі |
| `has_yearbuilt` | Прапорець: чи відомий рік побудови |
| `suburb_freq` | Частота зустрічання району (замість OHE з 300+ категоріями) |

### 4. Результати моделей (тестова вибірка 20%)

| Модель | MAE | RMSE | R² |
|---|---|---|---|
| Базова лінія (середнє) | 461 258 AUD | 630 259 AUD | 0.000 |
| Лінійна регресія | 256 183 AUD | 372 747 AUD | 0.650 |
| **Random Forest (100 дерев)** | **163 960 AUD** | **272 056 AUD** | **0.814** |

**Random Forest** найкращий: R² = 0.81, MAE = 164 тис. AUD  
(порівняно з базою 461 тис. — покращення у 2.8×)

### 5. Головні чинники ціни (за важливістю RF)
1. **Regionname_Southern Metropolitan** — район найбільше впливає на ціну
2. **Rooms** — кількість кімнат
3. **Distance** — відстань до центру
4. **Type_u** — тип юніт (дешевший)
5. **Landsize** — площа ділянки

## Висновки

- **Район і відстань до центру** — ключові чинники ціни в Мельбурні.
- **Лінійна залежність обмежена**: лінійна регресія пояснює лише 65% дисперсії, тоді як RF — 81%.
- **Пропуски несуть інформацію**: прапорці `has_buildingarea`/`has_yearbuilt` увійшли до ознак.

## Ідеї для покращення

1. **Таргетне кодування районів** (target encoding) замість частоти — може точніше відображати "престиж" району.
2. **Навчання на log(Price)** — симетричніший розподіл може покращити лінійну регресію.
3. **Gradient Boosting (XGBoost / LightGBM)** — як правило, перевершує Random Forest на табличних даних.
